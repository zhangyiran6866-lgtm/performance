# Zyr 的专属开发记忆与要求

这是专门为您 (`zyr`) 保留的持久化记忆规则文件（被定义为 AI Skill）。因为它是以特定的格式存在的，在以后每次新建对话或进行代码开发时，我都会在“自带技能库”里看到并读取这份文件的内容，从而严格遵循您的要求。

## 开发偏好清单

- **代码头部注释要求**：每次开发（新建或修改代码）时，必须在对应文件最上方（如果是修改则是本次功能逻辑块前）自动生成头部注释，注明：
  - **Author (`@author`)**: Zyr
  - **Date (`@date`)**: 本次开发的日期时间 (YYYY-MM-DD HH:mm:ss)
  - **Description (`@description`)**: 本次开发的功能概述
  - **Lines (`@lines`)**: 预估本次开发/变动的代码行数
- **代码内联注释要求**：书写业务代码时，如果存在复杂的代码模块或逻辑，必须使用**简明扼要的中文字符注释**注明功能概述。
- **Git 提交记录与推送偏好**：
  - 代码改动后，AI **不得擅自自动执行 git push 推送到远端**，所有操作必须根据用户直接的明确指令来进行。
  - AI 在准备 `git commit` 时，必须向用户**询问所需的 commit message**。如果在对话中用户指出需要 AI 自动生成，AI 应当生成**尽量简短**的内容。
- **自定义快捷指令（合并与提交流程）**：
  1. **“提交开发”**：代表需要将当前的 `zyr` 个人分支最新代码合并到 `dev` (开发分支)。如果在合并中出现代码冲突，AI 不要擅自解决，而是向用户如实报告并询问意见。
  2. **“提交测试”**：代表需要将 `zyr` 个人分支合并到 `test` (测试分支)。
  3. **“提交正式”**：代表需要将 `zyr` 个人分支合并到 `prod` (线上分支)。
  4. **“压缩上下文”**：收到此指令时，AI 必须：
     - 总结并提炼当前对话/开发阶段的全部核心关键变动。
     - 自动将这段提炼出的精简版更新内容以 Markdown 格式追加存储到项目根目录的 `CHANGELOG.md` 更新日志文件当中。

>
> 1. 每当在 `zyr` 环境/分支下进行代码开发时，任何操作和设计都**必须绝对服从**此文件上方所列的清单。

> 3. 不论何时，只要文件被更新，都需要准备 git 提交记录，但**必须遵守上方定义的 Git 提交流程**，不得擅自 push。
