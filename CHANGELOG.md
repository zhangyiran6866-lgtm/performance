## 2026-03-05 更新摘要

### 1. 网络配置与接口对接

- **环境直连**：将实际测试的后端目标地址 `VITE_API_TARGET_URL` 正式切换到 `http://192.168.10.108:9800`，舍弃本地 `proxy` 代理拦截，实现和后端的纯净直连。
- **Axios 拦截器还原**：100% 还原了 `c-center` 请求机制：
  - 在 `headers` 中正确附加 `tenant-id` 获取降级（由于本地测试，强制写入临时 '1'），并添加业务 `tag` 环境标志。
  - **Bugfix**：对包含中文字符的 `tenant-id` 添加了 `encodeURIComponent` 转码加密，彻底解决了浏览器报 `String contains non ISO-8859-1 code point` 参数集异常导致的请求崩溃问题。

### 2. 登录模块闭环 (Login.vue)

- **UI 及扫码框优化**：调整了 Login 登录页面自适应，修改 iframe 宽度、高度至 450px 并禁用原生双端滚动条（`scrolling="no"`），解决出现滚动条遮挡企业微信二维码等问题。
- **登录鉴权参数校验**：
  - 修复企业微信鉴权 `redirectUri` 授权 URL 的双重编码失效问题，还原了原有的 `socialAuthRedirect` 串接协议请求结构。
  - 解决了账号密码表单请求时需要用传统表单对象格式对标后台 `/system/oauth2/token` 接口的问题。

### 3. 全局路由防线机制

- **导航守卫拦截**：在 `router/index.ts` 搭建全局鉴权围墙 `router.beforeEach`。
  - 对于未登录的非法访问，一律退回 `/login` 并自动存续原始地址。
  - 成功持有校验票据在案后，若再次强制寻址 `/login` 将被强退弹回绩效配置中心页面（`/configuration`）。

### 4. 彻底接管状态与存储容器

- **引入体系化缓存库**：放弃原生的轻量 `localStorage`，安装并全量配置了 `pinia`, `pinia-plugin-persistedstate` 以及关键依赖 `web-storage-cache` 以完全对标 `c-center`。
- **确立全局键名统一结构**：
  - 对于 Token 双端拆离处理，存成 `ACCESS_TOKEN` 与 `REFRESH_TOKEN`。
  - 将用户数据（如 `permissions`, `roles`, `user` 等）提取，并统一入库为 `CACHE_KEY.USER`（即 `user`）通过 `useUserStoreWithOut().setUserInfoAction()` 同步加载挂载系统状态。
  - 对“记住密码”（`LoginForm`）实装回写/清除管理。

### 5. 多人联调机制与企业级体验防线升级

- **网关路由 Tag 支持**：在 `.env.local` 引入 `VITE_APP_SERVICE_TAG`，利用其配合 Vite 热更新，实现在 Axios `headers` 注入动态的开发人员机器标识符（Tag），无需重启应用即可与不同的后端人员即插即用完成接口定向分发联调。
- **重构路由降级体验**：当普通未授权访问被踢至 `/login` 页时，顶部追加了金黄色警告提示（`ElMessage.warning`）作“温和提醒”，避免用户面对页面跃迁茫然无措。
- **构建企业级“无感知 Token 刷新”防御网**：
  - 扩大 `401 Unauthorized` 拦截侦测范围：现已兼顾真实的网络协议 HTTP 401 与业务代码层（JSON body 中 `code === 401`）两大场景触发机制。
  - **无感并发重发阵列**：截获所有因超期触发的 401 后，在底层冻结挂起这些常规请求（进入排队队列），并于后门自动下发 `/system/auth/refresh-token` 换发全新 Token；认证成功后无缝回放队列恢复交互，从根本意义上保护了用户长时间停留应用的沉浸感。
  - 熔断机制完善：当底层最终令牌也确认失效时，配合 `isRelogin.show` 防并发弹窗锁定，确保有且仅弹出一次带选择的重新登录引流框（带防死锁清洗），彻底消除用户报错焦虑。

### 6. 字典配置与状态管理工具链整合升级

- **字典库初始化**：从原项目环境中成功接管并抽取补全了数据字典全链路状态工具体系（`src/utils/dict.ts` 及 `src/store/modules/dict.ts`），并对核心函数 `getDictOptions` 进行了前端方法层适配隔离，封装出标准通用获取通道机制。
- **业务配置灵活化落地**：将《指标配置元数据》组件 `IndicatorWizard.vue` 中的“所属维度分类”从固定的本地预制表彻底重构重写为远程/动态状态接入机制。如今改由通过 `classification_performance_indicators_type` 进行自动捕获渲染，实现了该表单选项的数据驱动和字典化。
