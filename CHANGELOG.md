  - 熔断机制完善：当底层最终令牌也确认失效时，配合 `isRelogin.show` 防并发弹窗锁定，确保有且仅弹出一次带选择的重新登录引流框（带防死锁清洗），彻底消除用户报错焦虑。

### 1. 考核模板编辑器 (Builder.vue) 交互与体验深度优化

  - **逻辑调整**: 移除了模拟的“全公司适用”选项，仅展示真实部门树；设置所有根节点为禁用状态（仅辅助展开，不可选中），确保考核方案精准下发至业务单元。
  - **展开策略**: 优化了部门树的初始展开逻辑，现在默认只展开第一级（根部）节点，减少视觉干扰。
  - **一键清空**: 增加了 `clearable` 功能，支持快速重置已选部门或人员。
- **选取人员组件全方位增强**:
  - **复选框模式**: 引入了选项前置复选框，替代原有的右侧对勾，操作感更强。
  - **快捷全选页脚**: 下拉列表底部增加了固定操作栏，支持一键“全选当前备选人员”和“取消全选”，并实时显示“已选/总计”的人数统计。
  - **联动自动选择**: 实现了“选择部门后自动全选该部门下所有人员”的功能，极大简化了全员考核场景的操作链路。
- **指标选取弹窗 (IndicatorSelectorModal.vue) 优化**:
  - **数据源转型**: 彻底摒弃 Mock 数据，接入 `getIndicatorList` API 动态获取系统指标库。
  - **视觉美化**: 指标分类标签改为动态背景色方案；调大了计分规则字段字号并加深颜色；标题栏增加了实时指标总数展示。
- **版块细节调优**:
  - 调大了“评估项配置”卡片中“绩效转化”和“目标对比”版块的提示文字及对比盒子的尺寸，增强视觉重心。
  - 显著放大了导航栏返回按钮的点击区域及其图标尺寸。

### 2. 发布流程安全性与状态管理闭环

- **前置强制校验机制**: 建立了发布前的“三级防御”逻辑。用户必须同时满足“指标权重合计 100%”、“已确认日报界面预览”以及“已保存最新草稿”这三个条件，发布按钮方可激活。
- **发布二次确认**: 增加了确认风险告知弹窗，明确提示用户“发布后生效周期内指标将无法修改”，由用户手动确认后方可提交。
- **保存状态感知**: 建立了全域数据深度监听。一旦对模板名称、人员、范围或指标详情产生任何改动，顶部“已保存”状态会即刻重置为“草稿中”警示，并提示“未保存更改”。
- **发布逻辑留存**: 将发布成功后的自动跳转改为原地留存，为后续接入“只读查看”模式预留逻辑空间。

### 3. 全局风格标准化标准化

- **组件样式对齐**: 强行同步了 `el-tree-select` (部门) 和 `el-select` (人员) 的高度至 `40px` (h-10)，确保在视觉比例上与标准考评频次下拉框实现水平对齐。
- **卡片布局优化**: 微调了全局 `Card` 组件的内部间距策略，将内容区的默认 `gap-6` 压缩至 `gap-2`，解决了业务项布局过于稀疏的问题。
